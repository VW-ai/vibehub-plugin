# Setup recovery

Use the typed error and affected path from the setup result. Preserve the exact
checkout and executable while recovering.

- **Non-Git target:** stop. Never run `git init`; ask the user to select or
  initialize the intended repository themselves.
- **Missing CLI:** repeat only the bounded resolution checks. Never download,
  install, build, or invent a launcher during project setup.
- **Managed marker conflict:** show the conflict and path. Do not insert, remove,
  reorder, or deduplicate managed markers by hand.
- **Symlink or non-regular instruction target:** stop and let the user resolve
  ownership. Do not follow or replace the symlink.
- **Unreadable or permission failure:** report the OS error and path. Do not
  chmod, elevate privileges, or replace the file without separate approval.
- **Partial apply or retained backup:** report which targets changed and the
  retained recovery path. Never delete a retained recovery backup
  automatically; preserve it until the user reviews recovery.
- **Repeated setup:** unchanged is success with no Effects. Do not rewrite
  current instruction blocks.
- **Restart or wrong worktree:** restart the host in the exact checkout and
  re-read status. Do not accept another worktree's session as Connected.
- **Empty query:** it proves no context value. Refine the real need or ingest
  authentic durable evidence; never manufacture a write.
- **Distill skipped or failed:** setup may remain Connected. Report the skip or
  failure honestly and leave Activated to future meaningful work.

After any user-approved repair, start again with inspect. Never skip directly to
apply based on stale evidence.
