# 06-workbench-app — Workbench App

The App is the visible product surface for understanding how VibeHub is working.
It is optional for the plugin runtime, but essential for dogfood, trust and
onboarding new users.

## Product boundary

- The App consumes stable browser-safe projections and typed receipts.
- It never reads SQLite tables as a second business-logic implementation.
- A future Git/YAML semantic backend must remain behind the same projection.
- The approved visual authority is the compact v8 spatial-workbench language.
- UI copy must distinguish captured evidence, queued work and persisted durable
  context.

## Immediate direction

After open source, build `06-01-live-shell` first so the team can run VibeHub in
real projects and see activation, live work and context memory behavior.
